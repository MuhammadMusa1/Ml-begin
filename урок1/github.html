1
Создание учётной записи GitHub

GitHub — это платформа для хранения кода и совместной работы. Все проекты курса будут храниться здесь.

Шаг 1. Переход на сайт

Откройте браузер (Chrome, Edge, Firefox). Перейдите по адресу:

https://github.com

Нажмите кнопку Sign up (справа вверху).

Шаг 2. Ввод данных

Заполните поля:

  • Email – действующий адрес (родительский или личный).
  • Password – минимум 8 символов, включая буквы и цифры.
  • Username – уникальное имя (латиница, цифры, дефис).
Важно: Имя пользователя должно быть профессиональным. Пример: ivan_petrov_ml или alina_2026. Не используйте клички или случайные наборы цифр.

Шаг 3. Подтверждение email

GitHub отправит письмо с кодом или ссылкой. Проверьте папку «Входящие» и «Спам». Нажмите Verify email address или введите код.

✅ После подтверждения – запишите логин (username), пароль и email в надёжное место. Сообщите родителям, что вы создали аккаунт для обучения.

Настройка профиля и присоединение к организации

Настройка профиля

После входа нажмите на иконку аватара (右上角) → Settings → Profile.

  • Добавьте реальное имя (Name) и фамилию.

Это поможет преподавателю идентифицировать вас.

Вступление в организацию курса

Перед первым занятием преподаватель отправит приглашение на вашу почту (связанную с GitHub). Письмо будет от github.com с темой «Invitation to join organization».

Действие: Откройте письмо, нажмите зелёную кнопку Accept Invitation. После этого вы станете участником организации, где будут опубликованы учебные репозитории и задания.

Если письмо не пришло в течение 10 минут после начала курса – проверьте папку «Спам» или сообщите преподавателю в чате.
3
Установка необходимого ПО (Windows / macOS / Linux)

1. Установка Git

Скачайте с официального сайта: git-scm.com/downloads. Выберите версию под вашу операционную систему.

Windows: запустите установщик, оставьте настройки по умолчанию (важно: выберите "Git from the command line and also from 3rd-party software").

macOS: можно через терминал: xcode-select --install или скачать установщик.

Проверка: Откройте терминал (CMD / PowerShell / Terminal) и введите:

git --version

Вы должны увидеть версию, например git version 2.42.0.

2. Установка Python

Сайт: python.org/downloads. Скачайте последнюю стабильную версию 3.11 или 3.12.

КРИТИЧЕСКИ ВАЖНО: При установке обязательно отметьте флажок Add Python to PATH. Иначе Python не будет работать в терминале.

После установки перезагрузите компьютер (рекомендуется). Проверка:

python --version

Должно отобразиться Python 3.11.x или выше.

3. Установка Visual Studio Code

Скачайте с code.visualstudio.com. В процессе установки на Windows отметьте следующие пункты:

  • ✓ Add "Open with Code" to Windows Explorer context menu
  • ✓ Add to PATH (важно для терминала)
  • ✓ Register Code as an editor for supported file types

После установки запустите VS Code.

4
Настройка VS Code и подключение к GitHub

Расширения (Extensions)

Откройте VS Code. Нажмите на значок расширений (Ctrl+Shift+X). Установите следующие:

  1. Python (от Microsoft) – поддержка Python и Jupyter.
  2. Jupyter – работа с ноутбуками.
  3. GitHub Pull Requests and Issues – интеграция с GitHub.
  4. GitHub Copilot – опционально (требуется подписка, но в учебных целях можно пропустить).

Авторизация GitHub в VS Code

В VS Code откройте панель "Accounts" (иконка шестерёнки внизу слева или через меню View → Command Palette).

Нажмите Sign in with GitHub. Откроется браузер, разрешите доступ. Вернитесь в VS Code – ваш профиль GitHub отобразится.

После входа вы сможете клонировать репозитории и делать commit/push прямо из редактора.
Совет: Для удобства откройте терминал в VS Code (Ctrl+`). Проверьте Git и Python ещё раз:
git --version
python --version
5
Настройка Git: имя пользователя и email

Чтобы коммиты подписывались вашим именем, выполните в терминале (VS Code или системный терминал):

git config --global user.name "ваше_имя_пользователя_github"
git config --global user.email "ваш_email@example.com"

Важно: email должен совпадать с тем, который привязан к GitHub.

Проверка настроек

git config --list

Вы увидите user.name и user.email. Это гарантирует, что ваши работы будут правильно подписаны.

Если вы не выполните эту настройку, GitHub не свяжет коммиты с вашим аккаунтом.

Диагностический скрипт (проверка окружения)

Создайте файл check.py со следующим содержимым и запустите его:

import sys, subprocess, platform print("Python версия:", sys.version) print("ОС:", platform.system(), platform.release()) git_ver = subprocess.getoutput("git --version") print("Git:", git_ver) print("Статус: OK — среда готова к курсу ML")

Запуск: в терминале перейдите в папку с файлом и выполните python check.py. Если ошибок нет – всё готово.

Для работы с API (OpenAI, Anthropic) в рамках уроков по LLM преподаватель предоставит учебные ключи или бесплатные альтернативы. Отдельный платёж не требуется.

Что делать, если что-то пошло не так

Частая проблема: python не распознаётся как внутренняя команда

Решение: Переустановите Python и обязательно отметьте "Add Python to PATH". После установки перезагрузите компьютер. Если ошибка остаётся – вручную добавьте путь в переменные среды (обратитесь к преподавателю).

Ошибка авторизации GitHub в VS Code

Убедитесь, что вы разрешили доступ в браузере. Попробуйте выйти из GitHub в VS Code и войти заново через команду GitHub: Sign out в палитре команд (Ctrl+Shift+P).

Git не установлен или неизвестная команда

Скачайте Git с официального сайта и установите. После установки перезапустите VS Code и терминал.

Правило: Если вы выполнили все шаги и что-то не работает – сделайте скриншот ошибки, опишите действия и отправьте в чат курса (или преподавателю). Мы поможем в течение 24 часов.
